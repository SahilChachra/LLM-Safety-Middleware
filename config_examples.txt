# Example Configuration Files
# ============================
#
# There are two separate config objects:
#   1. SafetyConfig  — controls the safety-check layers
#   2. LLMBackendConfig — controls the remote LLM backend
#
# SafetyConfig can be loaded from a JSON file (CONFIG_PATH env var or
# SafetyConfig.from_json("...")).  LLMBackendConfig is configured via
# environment variables (see api_server.py module docstring).

# ──────────────────────────────────────────────────────────────────────────
# config_development.json
# Development environment — lenient thresholds, verbose logging
# ──────────────────────────────────────────────────────────────────────────
{

# Notes on removed fields (v1 → v2 migration):
# The following SafetyConfig fields were removed in v2 when the pipeline was
# converted to a middleware proxy.  They are now configured through
# LLMBackendConfig / environment variables:
#   llm_model_name  → LLM_MODEL env var
#   max_new_tokens  → LLM_MAX_NEW_TOKENS env var
#   temperature     → LLM_TEMPERATURE env var
#   top_p           → LLM_TOP_P env var
#   batch_size      → removed (no local generation)
#   max_cache_size  → removed
